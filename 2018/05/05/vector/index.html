Vector

#介紹
可自行擴增大小的動態陣列

記得要引入<vector>標頭檔

##建立
有幾種常見的建立方法

  1. vector<type> name 複雜度：O(1)
    開一個名為name的新陣列，裡面裝的是type這個型別
  2. vector<type> name(a, b) 複雜度：O(n)
    開一個名為name的新陣列，裡面裝的是type這個型別，然後預先塞進a個b（b要屬於type）
  3. vector<type> name[a] 複雜度：O(n)（要開a次）
    開一個名為name的新二維vector，裡面裝的是type這個型別
    這等於開了a個一維vector，分別為name[0], name[1], …, name[a-1]

##增加

  1. name.push_back(a) 複雜度：O(1)
    在name的尾端放進一個a
  2. name.assign(a, b) 複雜度：O(n)
    把name裡面的元素全部清空（但不清掉已經佔用的記憶體）然後放進a個b
  3. name.reserve(a) 複雜度：O(n)
    把name所佔用的記憶體大小開到a。如果a小於原本的記憶體大小，不進行任何操作。
  4. name.insert(a, b) 複雜度：O(n)
    在a所指向的元素前面插入b（注意：a是指標！！）

##減少

  1. name.pop_back() 複雜度：O(1)
    把name最尾端的一個元素移除
  2. name.clear() 複雜度：O(n)
    把name清空（但不清掉已經佔用的記憶體）
  3. name.resize(a) 複雜度：O(n)
    把name的大小減到剩下a（但不清掉已經佔用的記憶體）
  4. name.erase(a) 複雜度：O(n)
    把a所指向的元素移除（注意：a是指標哦）

##存取

  1. name[a] 複雜度：O(1)
    回傳索引值為a的元素的索引
  2. name.front() 複雜度：O(1)
    回傳最前面的元素的索引（相當於name[0]）
  3. name.back() 複雜度：O(1)
    回傳最後面的元素的索引（相當於name[name的大小-1]）
  4. name.at() 複雜度：O(1)
    跟name[]一樣，差別在於name.at()比較安全，不會越界存取，但也比較慢。

##迭代

  1. name.begin() 複雜度：O(1)
    回傳最前面元素的指標
  2. name.end() 複雜度：O(1)
    回傳最後面元素的指標

##取得參數

  1. name.size() 複雜度：O(1)
    回傳name元素的總數
  2. name.capacity() 複雜度：O(1)
    回傳name所佔的記憶體大小
  3. name.empty() 複雜度：O(1)
    如果name是空的就回傳True
    要不然就回傳False

##特殊用法
vector<bool> name
這個型別的vector有特別優化過，每個元素只佔1bit
